# 관계형 데이터베이스(RDBMS)

관계형 데이터베이스(RDBMS)는 구조화된 데이터를 테이블 형태로 저장하고,
명확한 스키마와 제약조건을 기반으로 데이터 정합성을 강하게 보장하는 저장소이다.
트랜잭션 및 복잡한 질의 처리가 필요한 핵심 비즈니스 데이터를 관리하는 데 주로 사용된다.

---

## 1. 개념 및 특징

### 1.1 스키마 기반 데이터 모델
- 테이블, 컬럼, 타입이 사전에 정의됨
- 데이터 구조가 명확하고 예측 가능
- 도메인 모델을 직접적으로 표현 가능

### 1.2 관계와 무결성
- Primary Key, Foreign Key, UNIQUE, CHECK 등의 제약조건 제공
- 데이터 간 관계를 명시적으로 표현
- 데이터 오류를 DB 레벨에서 예방 가능

### 1.3 트랜잭션(ACID)
- Atomicity: 트랜잭션은 전부 수행되거나 전부 롤백됨
- Consistency: 트랜잭션 전후 데이터 일관성 유지
- Isolation: 동시성 환경에서도 트랜잭션 간 간섭 최소화
- Durability: 커밋된 데이터는 장애 발생 후에도 유지

### 1.4 풍부한 질의 기능
- JOIN, GROUP BY, ORDER BY, 서브쿼리 등 복잡한 조회 가능
- 통계, 분석, 리포팅에 적합

---

## 2. 장점

- 강력한 트랜잭션과 데이터 정합성 보장
- 복잡한 비즈니스 규칙 및 관계 표현 가능
- 표준화된 SQL 기반 생태계
- 백업, 복제, 모니터링 등 운영 도구 성숙

---

## 3. 단점

- 수평 확장(샤딩)이 구조적으로 복잡
- 대규모 트래픽 환경에서 스케일링 비용 증가
- 스키마 변경 시 마이그레이션 비용 발생

---

## 4. 설계 시 고려 사항

### 4.1 스키마 설계
- 비즈니스 규칙과 변경 가능성을 고려한 정규화/비정규화 전략
- 지나친 정규화로 인한 JOIN 비용 증가 주의

### 4.2 인덱스 전략
- 조회 패턴 기반으로 인덱스 설계
- 쓰기 성능과 조회 성능 간의 트레이드오프 고려

### 4.3 트랜잭션 경계
- 트랜잭션 범위를 최소화하여 락 지속 시간 감소
- 장시간 트랜잭션 지양

### 4.4 확장 전략
- Scale-up 한계 도달 시 샤딩, 리드 레플리카 고려
- 샤딩 시 키 선정과 쿼리 변경 비용 고려

---

## 5. 적합한 사용 사례

- 주문, 결제, 정산 시스템
- 회원 정보, 권한, 포인트 관리
- 강한 데이터 정합성이 요구되는 핵심 도메인
